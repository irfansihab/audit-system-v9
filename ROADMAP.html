1 Prinsip arsitektur (warisan v7)

  • backend/v6 READ-ONLY — semua perubahan di app-layer + template; V6 tak ditulis ulang.
  • 2 agen Claude (AT & KT); ingestion & QC SAIPI = deterministik/sinkron, bukan agen.
  • Skill registry folder-driven; skill = substansi domain, orkestrasi di prompt agen (Tahap R0–R4).
  • Anti-halusinasi: tiap temuan ber-dokumen_sumber; sebab hanya untuk skill AUDIT.

2 Prinsip UX — UI clean & minim friksi

Permintaan utama user: “UI clean, tidak banyak tombol atau hal yang membingungkan.” Ini pagar desain v8 — berlaku di setiap layar & fitur baru.
  • Satu aksi utama per layar — tombol primer jelas (maks. 2). Aksi lain di menu “⋯ / Lainnya”.
  • Progressive disclosure — opsi lanjutan/teknis disembunyikan; default cerdas mengurangi pilihan.
  • Tanpa tombol mati/duplikat — tiap tombol punya aksi nyata (tema & bell sudah dihapus di v7).
  • Navigasi tunggal lewat tahapan (ala SIMWAS) — auditor selalu tahu “ada di tahap mana”.
  • Bahasa manusia — label kata kerja jelas (“Susun PKP”, “Setujui Temuan”), bukan jargon/nama tool.
  • Status terlihat sekilas — progres & warna (hijau/kuning/merah) terbaca tanpa menebak.
  • Acceptance tiap fitur UI: “auditor non-teknis paham apa yang harus diklik dalam < 5 detik, tanpa pelatihan.”

Workstream

A

Konsolidasi identitas INTEGRAL tetap nama

Hapus dualitas (“Powered by Audit AI v7”), bukan ganti nama. “Audit AI” jadi engine/versi internal.

A1 — Hapus framing “Powered by Audit AI v7” dari UI (landing, TopBar, footer, meta title) → cukup “INTEGRAL — Workspace Pengawasan Inspektorat II”.
A2 — Identitas tetap: pertahankan logo ∫ + palette ungu #5C4FE7. Tidak ada aset baru.
A3 — Satukan narasi dokumen (README + HANDOVER) sebagai dokumen INTEGRAL tunggal.
A4 — Versi internal v7→v8 di config/package.json/badge, tanpa menyentuh brand.
A5 — Rapikan penamaan teknis (openapi-integral-v8) + buang referensi legacy audit-system-v4 di skill non-reviu.
A6 — Arsip docs rencana “fase INTEGRAL” usang ke docs/archive/.
A7 — Audit UI clean (lihat Prinsip UX): hapus tombol/kontrol membingungkan, terapkan “satu aksi utama per layar”; checklist UX per halaman jadi gate kelulusan fitur.
B

Autentikasi username + password baru · fondasi

Saat ini login = prototype role-only (pilih AT/KT/PT/PM tanpa password). v8 ganti ke kredensial nyata.

B1 — Skema: username (unik) + password_hash di tabel User; hash argon2/bcrypt (jangan plaintext).
B2 — Backend auth: login → JWT, seeding admin awal, ganti password. Klaim role tetap (RBAC AT/KT/PT/PM).
B3 — Frontend: form login username+password (ganti kartu peran), guard route, halaman ganti password.
B4 — Keamanan: rate-limit + lockout, kebijakan password, logout/expiry.
B5 — SSO SIMWAS koeksistensi: login lokal + SSO JWKS; produksi → SSO, dev → lokal.
C

Fitur dipertahankan (verifikasi pasca-konsolidasi)

C1 — Workflow 7-tahapan tetapKartu Penugasan → PKP → KKP (AI+HITL) → LRS KK → Konsep LHP → LRS LHP → Laporan. Pastikan status & panel approval/LHP tetap jalan.
C2 — Integrasi SIMWAS v2 tetapFinalisasi kontrak REST (openapi→v8), JWKS SSO, webhook; selaras dengan B5.
C3 — CACM/EWS tetapModul CACM + CacmRun/EwsFinding + halaman CACM dipertahankan & diverifikasi.
C4 — Mutu agen & eval lanjut v7Skill R0–R4 (reviu selesai) → lanjut rumpun audit/evaluasi/pemantauan; PKP-di-feedback; backend/eval.
C5 — TLHP sebagai pilar penuh baruNaikkan pemantauan-tindak-lanjut dari skill skeleton jadi modul: data model matriks rekomendasi + aging; dashboard TLHP (per satker/PIC, hijau→merah, kritis >365 hari) sebagai menu utama; auto-ingest rekomendasi dari laporan terbit (Tahapan 7) → menutup lingkaran; lengkapi references + agen.

Cara menjalankan v8 (lokal)

# 1. Database (Docker) — pakai compose yang sama
cd "sistem audit v8"
docker compose up -d db

# 2. Backend (install dependency — tidak ikut clone)
cd backend
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python -m uvicorn app.main:app --port 8000

# 3. Frontend
cd ../frontend
npm install
npm run dev
Catatan port: v7 & v8 memakai port (8000/3000) & DB (audit_v7) yang sama. Untuk jalan berdampingan, ubah port + DATABASE_URL→audit_v8 di .env v8. Untuk pindah total ke v8, matikan service v7 dulu.
